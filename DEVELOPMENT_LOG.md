## 项目概述

本项目是一款基于分级记忆法的单词学习 Web 应用。本阶段完成了核心功能的开发，包括单词卡片展示、四级分类、词库筛选、总体记忆配比、数据持久化、单词本导入等。后续将继续完善分组学习、发音等功能。

## 技术亮点

1. **工作列表机制**  
   标记单词时只修改原始数据，不刷新当前工作列表，用户可在当前列表中自由切换回溯，修正误标记。

2. **配比随机算法**  
   采用“先选等级，再选单词”方式，支持用户自定义“不认识:不太熟:认识”权重，按比例智能抽取复习单词。

3. **模块化代码**  
   按功能拆分 JavaScript 文件：`data.js`（初始数据）、`script.js`（主逻辑）、`review.js`（配比算法）、`storage.js`（localStorage 封装）、`import.js`（导入模块）。

4. **响应式设计**  
   使用 Flexbox/Grid 布局，通过媒体查询适配不同屏幕尺寸，电脑上卡片更大更舒展，手机上自动换行。

5. **数据持久化**  
   单词等级和配比设置自动保存到 localStorage，刷新或关闭浏览器后数据不丢失。

## 遇到的问题及解决方案

| 问题 | 解决方案 |
|------|----------|
| **卡片遮挡下方按钮** | 将卡片固定高度改为 `min-height`，增加 `.card-container` 的 `margin-bottom`，调整内边距。 |
| **标记单词后无法回溯修正** | 引入工作列表 `currentWorkList`，标记时只修改原始 `words` 数组，不刷新当前列表。用户仍可在原列表中前后切换。只有切换词库时才重新生成工作列表。 |
| **总体记忆模式下，下一个单词如何按配比随机抽取** | 实现 `getWordByRatio` 函数：先计算总权重，随机选择等级，再从该等级单词中随机选一个。若该等级无单词则降级到非空等级。 |
| **词库为空时页面不显示提示** | 在 `markWordLevel` 中，标记后计算当前词库剩余单词数。若为 0，立即调用 `updateWorkList()` 刷新工作列表（得到空数组），显示空状态提示。 |
| **导入单词本后刷新页面数据丢失** | 导入时调用 `saveWords` 将新单词保存到 localStorage；初始化时优先读取 localStorage，若无数据才使用 `data.js`。 |
| **卡片翻转后切换单词，新单词仍处于翻转状态** | 在 `displayWord` 函数中，切换单词后主动移除卡片的 `flipped` 类，强制恢复正面。 |
| **第一个单词点击“上一个”跳转到最后一个（循环）** | 修改 `prevWord`/`nextWord` 逻辑：索引超出边界时不循环，并可选给按钮闪烁提示。 |
